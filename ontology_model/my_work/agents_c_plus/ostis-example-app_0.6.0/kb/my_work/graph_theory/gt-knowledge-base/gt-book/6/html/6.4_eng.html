Finding the shortest path (Dijkstra)

Each vertex in V is comparable to the mark - the minimum known distance from this vertex to a. The algorithm works in steps - each step he "visited" a top and trying to reduce the mark. The algorithm ends when all vertices visited.

Initialization. The label of the vertex a is set equal to 0, the remaining vertices labels - infinity. This reflects the fact that the distance from a to the other vertices are not yet known. All vertex marked as unvisited.

Step algorithm. If all vertices are visited, the algorithm terminates. Otherwise, from the not yet visited vertices selected vertex u, having the minimum mark. We consider all possible routes, in which u is the penultimate paragraph. Top, which lead edges of u, is said to be neighbors of this vertex. For each neighbor vertex u, except where noted how the visited, consider a new path length equal to the sum of the current mark of u and the length of the edge connecting u with this neighbor. If the value is less than the length of the label neighbor, replace the label value received value length. Having considered all the neighbors, mark the vertex u as visits and repeat step aloritma.
